# 睡眠定时停止播放

## 目标

给 Melodex Web/PWA 播放器增加睡眠定时能力。用户可以启动一个倒计时,到期后停止播放;默认开启「播完整首歌后再停止」,避免睡前突然切断当前歌曲。

## 范围

- 仅改前端播放器,不新增后端接口。
- 偏好按设备存储在 `localStorage`,与音量、播放模式、播放进度的现有策略一致。
- 桌面底栏、桌面全屏播放页、移动全屏播放页都能查看或调整定时状态。
- 不改变搜索、下载、缓存到本机、Subsonic facade 行为。

## 用户行为

- 播放器提供睡眠定时入口,可选择常用时长并启动。
- 启动后显示剩余时间。
- 用户可取消当前定时。
- 「播完整首歌后停止」开关默认开启。
- 开关开启时:
  - 倒计时到期后进入待停止状态。
  - 当前歌曲自然播放结束时停止,不再按列表循环、随机或单曲循环继续播放。
- 开关关闭时:
  - 倒计时到期立即暂停当前播放。

## 状态机

| 状态 | 条件 | 行为 |
| --- | --- | --- |
| idle | 未设置定时 | 不影响播放 |
| running | 已设置 `endsAt` 且未到期 | 显示剩余时间 |
| pendingEndOfTrack | 已到期且开关开启 | 保持当前歌播放,自然 ended 时停止 |
| expiredImmediate | 已到期且开关关闭 | 立即暂停并清理定时 |

## 验收标准

1. `npm run test:player` 覆盖睡眠定时纯逻辑。
2. `npm run build` 通过。
3. 浏览器验证桌面与移动视口:
   - 默认开关为开启。
   - 启动 1 分钟定时后可见剩余时间。
   - 取消后状态回到未设置。
   - 开关开启时,到期后自然 ended 不自动下一首。
   - 开关关闭时,到期立即暂停。

## Review 检查点

- 不硬编码路径、密钥、URL、端口或环境相关配置。
- 不破坏现有播放模式:未设置定时时,`order`、`loop`、`repeat`、`shuffle` 行为不变。
- 定时清理不会泄漏 `setInterval`。
- MediaSession `playbackState` 与播放器暂停状态保持一致。
